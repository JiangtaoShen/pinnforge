# b00 — control: baseline measured as-is (setup block, not a subagent)

## Thrust & rationale
b00 is the setup/control node, not a block product: it measures
`task/baseline.py` (the vanilla PINN tree root) exactly as shipped, to fix
the rRMSE every later block must beat. No fork — this *is* the root.

The baseline is a deliberately barebones vanilla PINN for the **KS-chaotic**
task (scaled Kuramoto–Sivashinsky, `u_t + V1·u·u_x + V2·u_xx + V3·u_xxxx = 0`
on `(t,x) ∈ [0,0.4]×[0,2π]`, periodic in x; score = relative L2 error on `u`
over the full reference grid):

- single plain MLP, 5×100, tanh everywhere, He-uniform init, scalar output
  `u`; **no** feature mapping, Fourier embedding, skip connections, hard BCs,
  or special activations (all left for blocks to discover);
- soft losses at equal weight (`WEIGHT_IC = WEIGHT_BC = 1`): PDE residual
  (KS, all derivatives by nested autograd incl. `u_xxxx`) + IC L2 against
  `ks_ic.csv` + periodic-BC L2 `u(t,0) = u(t,2π)`;
- collocation is an analytic interior grid (t=0 IC line excluded); IC points
  from `ks_ic.csv` (the only field data a candidate may read); no resampling;
- fixed LR 1e-3 Adam, no schedule; minibatched;
- wall-deadline training: whole process ≤ `TRAIN_TIME = 300 s`, stops
  `SAVE_MARGIN_S = 5 s` early.

## Findings
Baseline rRMSE **0.7389** (MSE 0.2564) — the floor to beat. Scoring is now
**plain relative L2 over the whole `t≤0.4` grid, no boundary override**
(aligned with the Scale-PINN and jaxpi reference protocols; the earlier
override of the domain edges — including the hardest final-time `t=0.4`
slice — was removed, which is why this number is far above the old 0.26).

The vanilla baseline leaves every standard PINN lever on the table: input
feature mapping / Fourier embeddings (kb1 020), PDE-vs-BC/IC loss weighting or
self-adaptive weights (kb1 023), causal / curriculum time weighting (kb1 084)
— essential for chaotic long-time error growth — better optimizers
(L-BFGS / SOAP / second-order, kb1 063/074), and throughput (300 s hard wall,
steps/sec matters). Consult `kb1/INDEX.md` for the full corpus.
